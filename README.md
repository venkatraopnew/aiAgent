## AI Recruitment Phone Interviewer (MVP)

Backend-first scaffold for an AI recruitment app that can:

- Create an interview session (from Flutter or any client)
- Start an outbound phone call (Twilio)
- Drive a spoken interview loop:
  - Candidate speaks (Twilio Speech Recognition via `<Gather input="speech">`)
  - Text goes to OpenAI (decide next prompt)
  - Prompt goes to ElevenLabs (TTS)
  - Audio is played back to the candidate via Twilio `<Play>`
- Persist a lightweight transcript in memory (swap to DB later)

### Architecture

Flutter App → FastAPI → OpenAI → ElevenLabs → Twilio → Candidate phone

### Prereqs

- Python 3.11+
- Twilio account + voice-capable phone number
- OpenAI API key
- ElevenLabs API key + voice id

### Setup

1. Create a virtual environment and install dependencies:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

2. Create `.env` (see `.env.example`):

```bash
cp .env.example .env
```

3. Run the server:

```bash
uvicorn app.main:app --reload --port 8000
```

4. Expose locally for Twilio webhooks (pick one):

- **ngrok**:

```bash
ngrok http 8000
```

Copy the HTTPS URL and set your Twilio phone number Voice webhook to:
`https://<your-ngrok-domain>/twilio/voice`

### Usage (from Flutter / Postman)

1. Create an interview:

`POST /api/interviews`

```json
{
  "candidate_name": "Aparna",
  "candidate_phone": "+15551234567",
  "role": "Customer Support Associate",
  "language": "en",
  "intro": "Hi! This is the first round interview. Are you ready to begin?"
}
```

Response includes `interview_id`.

2. Start a phone call:

`POST /api/interviews/{interview_id}/start_call`

### Notes / Limits (MVP)

- Uses Twilio `<Gather input="speech">` for speech-to-text (simple, not real-time streaming).
- Stores state in memory (resets when server restarts). Replace with Postgres/Redis later.
- Audio is generated by ElevenLabs per turn and stored under `./storage/audio/`.
